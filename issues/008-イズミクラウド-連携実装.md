# イズミクラウド: IZUMI 連携機能の実装

## 1. 概要 (Overview)

### 背景 (Background)

* **現状の課題:** 車両損益計算システム（IZUMI）は sync API を提供しているが、イズミクラウド側で連携クライアントが未実装のため、マスタデータ・経理データの連携が行われていない。
* **ビジネス要求:** イズミクラウドを車両損益計算システムの中心的な連携元として、マスタデータ・経理データの集約と送信を担当させること。
* **ユーザーストーリー:** システム管理者として、私はイズミクラウドから車両・コース・ドライバー・経理データを IZUMI に自動連携し、車両損益計算書を構築したい。

### 達成目標 (Goal)

* **あるべき姿:** イズミクラウドが IZUMI の sync API を呼び出し、マスタデータ・経理データを適切なタイミングで送信し、車両損益計算書が自動構築されること。
* **完了条件 (Definition of Done):**
* [ ] 認証・権限（JWT 取得、連携用ユーザー、トークン管理）が実装されていること。
* [ ] マスタデータ連携（users、courses、vehicles、vehicle-monthly-costs、location-monthly-expenses、drivers）が実装されていること。
* [ ] トランザクションデータ連携（import、records/bulk）が実装されていること。
* [ ] PCA・ATMTC・ITP からのデータ集約と送信が実装されていること。
* [ ] 部門 ID の統一、エラーハンドリング、スケジュール実行が整備されていること。

---
## 2. 仕様 (Specification)

### 機能要件 (Functional Requirements)

**認証・権限**
* `POST /api/auth/login` で `userId`（または `email`）と `password` を送信し、JWT を取得すること。
* sync API 利用には MASTER 権限（DX または DX管理者ロール）が必要。連携専用ユーザーを作成し、権限を付与すること。
* JWT 有効期限 7 日。期限切れ前に再取得する仕組みを実装すること。

**マスタデータ連携**
* POST /api/users/sync: ユーザーマスタを JSON 形式で送信。`userId`（externalId）をキーに upsert。実行タイミング: ユーザー登録・更新時
* POST /api/courses/sync: コースマスタ送信。`departmentId`、`code`、`name`、`externalId` を含める。実行タイミング: コース登録・更新時
* POST /api/vehicles/sync: 車両マスタ送信。`courseId` / `courseExternalId` でコースと紐づけ。実行タイミング: 車両登録・更新時（コース sync の後）
* POST /api/vehicle-monthly-costs/sync: 車両別・年月別の固定費用。ITP 由来の `fuelEfficiency`・`roadUsageFee` も含める。実行タイミング: 月次（月初など）
* POST /api/location-monthly-expenses/sync: PCA から取得した拠点別月額経費（20 科目）を送信。実行タイミング: 月次（PCA 確定後）
* POST /api/drivers/sync: ドライバーマスタ送信。ATMTC のドライバー・コース紐づきを反映。実行タイミング: ドライバー登録・更新時、ATMTC 連携後

**トランザクションデータ連携**
* POST /api/import: CSV/Excel を multipart/form-data で送信。実行タイミング: 経理データ確定後（月次）
* POST /api/income-statement/records/bulk: 月次損益を JSON で一括登録。実行タイミング: 経理データ確定後（月次）

**他システムからのデータ集約**
* PCA: 拠点別月額経費（20 科目）を取得し、location-monthly-expenses/sync で送信
* ATMTC: ドライバー・コース紐づきを取得し、drivers sync や courses sync に反映
* ITP: 燃費・道路使用料を取得し、vehicle-monthly-costs/sync の各車両データに含める

**その他**
* 部門 ID の統一（[department-id-standard.md](../docs/department-id-standard.md) 参照）
* 連携成功時に POST /api/sync-logs で記録（任意）
* 400/401/403/500 のレスポンスを処理し、リトライやアラートを検討
* マスタ・月次データの sync をバッチで定期実行する仕組み

# Izumi Cloud: Triển khai chức năng liên kết IZUMI

## 1. Tổng quan

### Bối cảnh

* **Vấn đề hiện tại:** Hệ thống tính toán lãi lỗ xe (IZUMI) cung cấp sync API nhưng phía Izumi Cloud chưa triển khai client liên kết, do đó việc liên kết dữ liệu master và dữ liệu kế toán chưa được thực hiện.
* **Yêu cầu kinh doanh:** Izumi Cloud đóng vai trò nguồn liên kết trung tâm của hệ thống tính toán lãi lỗ xe, chịu trách nhiệm tổng hợp và gửi dữ liệu master cùng dữ liệu kế toán.
* **Câu chuyện người dùng:** Là quản trị viên hệ thống, tôi muốn tự động liên kết xe, khóa học, tài xế và dữ liệu kế toán từ Izumi Cloud sang IZUMI để xây dựng báo cáo lãi lỗ xe.

### Mục tiêu đạt được

* **Hình ảnh lý tưởng:** Izumi Cloud gọi sync API của IZUMI và gửi dữ liệu master cùng dữ liệu kế toán đúng thời điểm, báo cáo lãi lỗ xe được xây dựng tự động.
* **Điều kiện hoàn thành (Definition of Done):**
* [ ] Xác thực và quyền (lấy JWT, người dùng liên kết, quản lý token) được triển khai.
* [ ] Liên kết dữ liệu master (users, courses, vehicles, vehicle-monthly-costs, location-monthly-expenses, drivers) được triển khai.
* [ ] Liên kết dữ liệu giao dịch (import, records/bulk) được triển khai.
* [ ] Tổng hợp và gửi dữ liệu từ PCA, ATMTC, ITP được triển khai.
* [ ] Thống nhất department ID, xử lý lỗi, thực thi theo lịch được hoàn thiện.

---
## 2. Thông số kỹ thuật (Specification)

### Yêu cầu chức năng (Functional Requirements)

**Xác thực và quyền**
* Gửi `userId` (hoặc `email`) và `password` qua `POST /api/auth/login` để lấy JWT.
* sync API yêu cầu quyền MASTER (role DX hoặc DX管理者). Tạo người dùng chuyên liên kết và cấp quyền.
* JWT có hiệu lực 7 ngày. Triển khai cơ chế lấy lại trước khi hết hạn.

**Liên kết dữ liệu master**
* POST /api/users/sync: Gửi master người dùng dạng JSON. Upsert theo key `userId` (externalId). Thời điểm: khi đăng ký/cập nhật người dùng
* POST /api/courses/sync: Gửi master khóa học. Bao gồm `departmentId`, `code`, `name`, `externalId`. Thời điểm: khi đăng ký/cập nhật khóa học
* POST /api/vehicles/sync: Gửi master xe. Liên kết khóa học qua `courseId` / `courseExternalId`. Thời điểm: khi đăng ký/cập nhật xe (sau courses sync)
* POST /api/vehicle-monthly-costs/sync: Chi phí cố định theo xe, năm tháng. Bao gồm `fuelEfficiency`, `roadUsageFee` từ ITP. Thời điểm: hàng tháng (đầu tháng)
* POST /api/location-monthly-expenses/sync: Gửi chi phí hàng tháng theo địa điểm (20 khoản mục) lấy từ PCA. Thời điểm: hàng tháng (sau khi PCA xác nhận)
* POST /api/drivers/sync: Gửi master tài xế. Phản ánh liên kết tài xế-khóa học từ ATMTC. Thời điểm: khi đăng ký/cập nhật tài xế, sau ATMTC

**Liên kết dữ liệu giao dịch**
* POST /api/import: Gửi CSV/Excel qua multipart/form-data. Thời điểm: sau khi xác nhận dữ liệu kế toán (hàng tháng)
* POST /api/income-statement/records/bulk: Đăng ký hàng loạt lãi lỗ hàng tháng dạng JSON. Thời điểm: sau khi xác nhận dữ liệu kế toán (hàng tháng)

**Tổng hợp dữ liệu từ hệ thống khác**
* PCA: Lấy chi phí hàng tháng theo địa điểm (20 khoản mục), gửi qua location-monthly-expenses/sync
* ATMTC: Lấy liên kết tài xế-khóa học, phản ánh vào drivers sync và courses sync
* ITP: Lấy燃費・道路使用料, đưa vào dữ liệu từng xe trong vehicle-monthly-costs/sync

**Khác**
* Thống nhất department ID (tham chiếu [department-id-standard.md](../docs/department-id-standard.md))
* Ghi nhận qua POST /api/sync-logs khi liên kết thành công (tùy chọn)
* Xử lý phản hồi 400/401/403/500, xem xét thử lại và cảnh báo
* Cơ chế chạy batch định kỳ cho sync master và dữ liệu hàng tháng
